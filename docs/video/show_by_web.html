核心概念与横向扩展

四大关键模块的技术解析

🧠

ReAct 推理模式

ReAct + Plan-and-Execute

单步 ReAct

每轮只返回一个步骤，强依赖上一轮 observation，适合高不确定任务（网页抓取、复杂检索）

多步计划模式

一次返回 JSON 数组，后端按序执行，适合结构化任务（"先读文件再汇总"）

{
  "thought": "分析当前状态...",
  "action": "调用工具...",
  "observation": "工具返回结果...",
  "final": "最终答案（可选）"
}

💡 扩展建议：动态模式切换策略，根据任务类型或历史失败率选择模式

🔌

MCP 协议机制

stdio + JSON-RPC

1. 启动子进程（command + args）
2. 发送 Content-Length 头的 JSON-RPC
3. 调用 tools/list 获取工具定义
4. 调用 tools/call 执行工具

工具系统

Schema 驱动 + 统一调度

JSON Schema 描述

工具参数使用 JSON Schema 描述并存储在 DB，模型可基于 schema 做更稳定的参数构造

统一调度

Agent 不关心工具来源（内置 or 外部 MCP），统一由 mcp_tools.py 分发

工具治理

开关、审计、限流可以在调度层集中处理，新工具接入成本低
